## 七、数据流关系

```
B站视频/音频/转写（01_原始资料）
        │
        ├──→ timeline.jsonl（时序索引）
        ├──→ audio_transcript_mapping.jsonl（音频-转写映射）
        ├──→ extracted_dates.jsonl（日期索引）
        │
        ├──→ 模糊词汇_详细.json（Whisper错误/暗号发现）
        │       └──→ 待确认关键词_详细.csv（人工确认后的映射表）
        │               └──→ 提及股票频率_修正版.csv（含暗号的频率统计）
        │
        ├──→ 决策案例库.json（操作记录提取）
        │       └──→ 训练数据集_完整版.json（AI训练数据）
        │
        └──→ events.jsonl + 莫大提及事件统计.json（事件关联）
```

## 八、维护备忘

### 新增视频时需要更新的文件：
1. `timeline.jsonl` — 追加新条目
2. `audio_transcript_mapping.jsonl` — 追加音频-转写映射
3. `extracted_dates.jsonl` — 追加有日期的条目
4. `待确认关键词_详细.csv` — 用Numbers打开，追加新暗号
5. `提及股票频率_修正版.csv` — 更新提及次数
6. `决策案例库.json` — 如有买卖操作，追加条目

## 十一、转写后处理工作流（标准流程）

每次新增视频转写后，按以下步骤处理：

1. **下载音频**：`yt-dlp --cookies ~/.bili_cookies_netscape.txt -x --audio-format mp3`
2. **Whisper转写**：`KMP_DUPLICATE_LIB_OK=TRUE whisper "音频.mp3" --model base --output_format txt --language zh --device cpu`
3. **原始转写**：存入 `01_原始资料/B站转写/`
4. **复制可读版**：复制到 `01_原始资料/B站转写_可读版/`
5. **批量修正**：
   - 先查本文件第二节的错误表，执行已知替换
   - 再查第十节易错词汇表，grep检查是否被误识别
   - 最后通读抽查，发现新模式后追加到本文件
6. **更新索引**：更新 timeline.jsonl、日期索引、频率统计等
