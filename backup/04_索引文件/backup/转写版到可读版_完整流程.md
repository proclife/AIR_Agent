## 二、Step 1 — B站视频下载（yt-dlp）

### 命令

```bash
yt-dlp \
  --cookies ~/.bili_cookies_netscape.txt \
  -x --audio-format mp3 \
  -o "01_原始资料/B站音频/%(title)s.%(ext)s" \
  "https://www.bilibili.com/video/BV号"
```

### Cookie

- 充电视频必须用Cookie（SESSDATA、bili_jct、buvid3）
- 路径：`~/.bili_cookies.json`（JSON格式）→ `~/.bili_cookies_netscape.txt`（Netscape格式）
- 一次性转换（见MEMORY.md中的转换脚本）
- 普通视频可省略 `--cookies`

## 三、Step 2 — Whisper语音转写

### 命令

```bash
KMP_DUPLICATE_LIB_OK=TRUE whisper "音频文件.mp3" \
  --model base \
  --output_format txt \
  --output_dir "01_原始资料/B站转写/" \
  --language zh \
  --device cpu
```

### 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `KMP_DUPLICATE_LIB_OK=TRUE` | 环境变量 | macOS必须，解决OpenMP冲突 |
| `--model` | `base` | 139MB，CPU推理最佳平衡 |
| `--language` | `zh` | 中文 |
| `--device` | `cpu` | Mac CPU模式 |

### Whisper已知局限性

- 中文音近字误识别严重（见下文修正列表）
- 无标点、无分段，输出为连续文本
- 数字识别不准（如"3.700万"、"88640.886414万"）
- `base`模型在CPU上约8分钟转78分钟音频
- `medium`/`small`模型CPU太慢，不推荐
- `faster-whisper`大模型内存不足会被SIGTERM杀

#### 4.2 Cascade污染问题（已解决）

`sed`链式替换"藏框/藏矿→西藏矿业"产生畸形字符串（西西藏矿业业液/业叶/业益/业艳等）。

**解决方案**：Python一次性正则+隔离cascade清理规则，200+残留归零。

## 六、关键技术要点

### 6.1 级联（Cascade）防护

替换时最关键的技术问题是**级联污染**——替换后的文本产生新的匹配目标，导致重复替换。

**通用规则**：
1. 先清理已知级联产物（如"液冷液冷→液冷"、"西西藏矿业→西藏矿业"）
2. 再用负向前瞻/后顾做主替换
3. 最后再清理一轮级联

**常用regex技巧**：

| 场景 | Pattern | 效果 |
|------|---------|------|
| 避免重复前缀 | `(?<!AI)液冷` | 不匹配"AI液冷"中的"液冷" |
| 避免重复后缀 | `航天(?!动力)` | 不匹配"航天动力"中的"航天" |
| 避免子串替换 | `(?<!西)藏矿` | 不匹配"西藏矿业"中的"藏矿" |
| 避免词尾匹配 | `(?<![气])储能` | 不匹配"智光电气储能"中的"储能" |

### 6.2 两阶段执行

所有批量替换脚本都采用两阶段：
1. **Phase 1**: 字符串替换清理所有已知级联产物
2. **Phase 2**: 正则替换执行主替换规则
3. **Phase 3**: 再次字符串替换清理新产生的级联
