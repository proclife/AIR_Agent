## 📊 核心索引文件（JSONL格式，可追加）

### timeline.jsonl
- **作用**：所有转写文件的时序索引，每行一条记录
- **字段**：`filename`, `date`, `size`, `word_count`
- **用途**：按时间线检索视频/转写内容，追踪内容发布时间
- **注意**：`date` 可能为 null（文件名中无法提取日期时）

### audio_transcript_mapping.jsonl
- **作用**：音频文件与转写文件的映射关系
- **字段**：`audio`, `transcript`, `date`, `audio_size`
- **用途**：确认哪些音频已有对应转写，发现缺失的转写
- **注意**：音频108个 vs 转写209个，差异来自纯文字动态

### extracted_dates.jsonl
- **作用**：已成功提取日期的文件索引
- **字段**：`filename`, `date`, `has_audio`
- **用途**：快速查询某日期范围内的内容

### events.jsonl
- **作用**：重大市场/宏观事件时间线
- **字段**：`date`, `event`, `type`(macro/sector), `impact`
- **用途**：关联莫大韭菜的观点与市场事件的对应关系

## 📝 维护指南

### 新增视频时需要更新的文件

1. **timeline.jsonl** — 追加新条目（filename, date, size, word_count）
2. **audio_transcript_mapping.jsonl** — 追加音频-转写映射
3. **extracted_dates.jsonl** — 追加有日期的条目
4. **待确认关键词_详细.csv** — 用 Numbers 打开，追加新发现的暗号
5. **提及股票频率_修正版.csv** — 更新提及次数
6. **决策案例库.json** — 如有买卖操作记录，追加条目

### 注意事项

- JSONL 文件用追加方式更新，不要覆盖
- `待确认关键词_详细.csv` 只能用 Numbers 编辑，不要用文本工具写入
- 新增暗号时同步更新 `03_分析报告/莫大韭菜_持仓与股票代号.md`
- 训练数据集需要人工审核后更新
